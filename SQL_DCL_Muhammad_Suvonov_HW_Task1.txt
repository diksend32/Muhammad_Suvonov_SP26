2 Authentication methods

Authentication methods are used to verify a user’s identity before allowing a connection to PostgreSQL, using methods such as password, OS-based authentication, or certificates.

We use pg_hba.conf to define and manage authentication methods in PostgreSQL.

To open it I used powershell:  notepad "C:/Program Files/PostgreSQL/18/data/pg_hba.conf"

# TYPE  DATABASE        USER            ADDRESS                 METHOD

# "local" is for Unix domain socket connections only
local   all             all                                     scram-sha-256 --> password required(authentication methods)  --- Uses strong hashing
# IPv4 local connections:
host    all             all             127.0.0.1/32            scram-sha-256
# IPv6 local connections:
host    all             all             ::1/128                 scram-sha-256
# Allow replication connections from localhost, by a user with the
# replication privilege.
local   replication     all                                     scram-sha-256
host    replication     all             127.0.0.1/32            scram-sha-256
host    replication     all             ::1/128                 scram-sha-256

Other authentication methods:
md5 - Password-based authentication, uses MD5 hashing
password - Sends password in plain text insecure
trust - No password required, anyone matching the rule can connect
..


Explain whether it is sufficient or risky. Identify at least one potential security vulnerability and suggest an improvement

Unlimited number of connections at the same time can be dangerous.

Each connection uses:memory CPU backend process ---> Too many connections can: slow down the database crash PostgreSQL freeze the system

We can change limit the  connections depending on conditions. If the database stores the special data from a one company we can put the  number of workers as limit limit
```
ALTER DATABASE dvdrental_dml CONNECTION LIMIT 100;
```
